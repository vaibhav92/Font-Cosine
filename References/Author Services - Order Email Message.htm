Order Email Messages for FontCosine

This screen will allow you to define custom order email messages that will be sent to the user after the purchase button is hit. 1 of 3 things will happen after the user hits the purchase button.
  1. The order will pass all internal checks and pass the credit card authorization - this order is Coded as Authorized
  2. The order will pass all internal checks and fail the credit card authorization - this order is Coded as Declined
  3. The order will is submitted by the user for Credit Card authorization - this order is Coded as Submitted Order
You can create a custom email message from one of these codes. Each order that is processed is assigned a tracking number (assiged at the point of purchase) and an order id number (assiged at the point of Authorization hence only authorized orders get and orderid number)

You can view what each of the standard email messages and what you can customize by clicking here.

The following variables are also available for you to put into the email message. They will be replaced with the actual value. For example, If the User entered in John Smith as the name on the order form - you can use $ordername to have it print out in the email message.
